Dependency Inversion Principle

- Зависимость от Абстракций. Нет зависимости от реализаций
- Абстракции не должны зависеть от деталей. Детали должны зависеть от абстракций – создаем интерфейсы, абстрактные классы, и обычные классы – идем от общего к частному
  Например, в интерфейсе нужно стараться использовать типы других интерфейсов, а не конкретные типы (абстрактное ссылается на абстрактное).
